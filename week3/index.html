▌ FGSM ATTACK
Fast Gradient Sign Method — single-step adversarial perturbation.
// WHAT IS FGSM?

The Fast Gradient Sign Method crafts an adversarial example in a single step. It computes the gradient of the loss with respect to the input image — essentially asking "which direction should each pixel move to maximally confuse the model?" — then nudges every pixel by exactly ε in that direction.

ε (EPSILON)
Controls perturbation size. Too small and the model shrugs it off. Too large and the image looks visibly distorted to humans. Your goal: find the point where the model breaks but the image still looks normal.
WHY DOES IT WORK?
Neural networks are locally linear in high dimensions — a tiny coordinated push across all input pixels accumulates into a large shift in output logits, flipping the prediction entirely.

▌ PGD ATTACK
Projected Gradient Descent — iterative, stronger attack.
// WHAT IS PGD?

Projected Gradient Descent is FGSM repeated many times. Each step takes a small gradient step, then "projects" back into the allowed perturbation budget. It finds a much stronger adversarial example because it iteratively searches for the worst-case input rather than taking one blind leap.

ε vs α vs STEPS
ε is the total budget — how far the final image can drift from the original. α is the step size per iteration; overshoot and you bounce around, too small and you converge slowly. Steps — more iterations find stronger adversarial examples, but with diminishing returns.
FGSM vs PGD
FGSM is one big step in the gradient direction. PGD is many small steps that explore the neighbourhood. PGD almost always succeeds where FGSM fails — it is considered the gold standard benchmark for evaluating adversarial robustness.

▌ MODEL EXTRACTION
Black-box attack — bypass the rate limiter to steal the victim model.
// WHAT IS MODEL EXTRACTION?

You have no access to the model's weights or training data — only its prediction API. By sending inputs and recording outputs, you can train a surrogate model that mimics the victim's behaviour. This is how real attackers steal proprietary models from cloud APIs.

THE RATE LIMITER
Real APIs monitor query volume and flag unusual patterns — systematic high-volume querying is a dead giveaway. You need to throttle your requests to blend in with normal traffic. Too slow and the extraction takes forever. Too fast and you get caught.
CLONE FIDELITY
This measures how closely your surrogate matches the victim's predictions — not just its general accuracy. High fidelity means you have effectively stolen the model's decision boundary, not just its approximate skill level.
